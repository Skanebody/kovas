## Top 10 issues à fixer — toutes traitées

### 1. [Tables — 6 fichiers] — `<th>` sans `scope="col"`
- **Critère RGAA** : 5.6 (Tableau de données — chaque en-tête de tableau a un attribut `scope`)
- **WCAG** : 1.3.1 (Info and Relationships)
- **Sévérité** : MAJEURE
- **Cause** : Les `<th>` rendent l'association ligne/colonne ambiguë pour les lecteurs d'écran
- **Fichiers** :
  - `apps/web/src/app/dashboard/dossiers/[id]/defense/page.tsx` (3 `<th>`)
  - `apps/web/src/app/dashboard/account/account-settings-client.tsx` (6 `<th>` table factures)
  - `apps/web/src/app/dashboard/account/parrainage/page.tsx` (3 `<th>`)
  - `apps/web/src/app/dashboard/analytics/benchmark-comparison.tsx` (6 `<th>`)
  - `apps/web/src/app/dashboard/dossiers/[id]/mission/tchat/MissionRecapSheet.tsx` (1+N `<th>`)
- **Fix** : Ajout `scope="col"` à tous les `<th>` d'en-tête de colonne

### 2. [`clients/[id]/page.tsx`] — Contraste insuffisant "Non renseigné"
- **Critère RGAA** : 3.2 (Contraste — texte normal sur fond clair)
- **Sévérité** : MAJEURE
- **Cause** : `text-sm text-[#0F1419]/55 italic` sur `bg-paper` ≈ 4.0:1 (en-dessous AA 4.5:1)
- **Fix** : `/55` → `/72` (ratio ~5.6:1)

### 3. [`account/user-hero-card.tsx`] — Email faiblement contrasté
- **Critère RGAA** : 3.2
- **Sévérité** : MAJEURE
- **Cause** : `text-[13px] text-[#0F1419]/55` (texte normal 13px → 4.5:1 requis, mesuré ~4.0:1)
- **Fix** : `/55` → `/72`

### 4. [`account/notification-prefs-form.tsx`] — Helper text contraste
- **Critère RGAA** : 3.2
- **Sévérité** : MAJEURE
- **Cause** : `text-[12px] text-[#0F1419]/55` paragraphe info
- **Fix** : `/55` → `/72`

### 5. [`account/settings-section.tsx`] — Header + helperText
- **Critère RGAA** : 3.2
- **Sévérité** : MAJEURE
- **Cause** : `h2 font-mono text-[11px] /55` + `p text-[12px] /55 leading-relaxed`
- **Fix** : `/55` → `/72` (2 occurrences dans le même fichier)

### 6. [`account/account-settings-client.tsx`] — Eyebrow "Identité" + thead tableau
- **Critère RGAA** : 3.2
- **Sévérité** : MAJEURE
- **Cause** : Eyebrow `font-mono /55` + thead `text-[#0F1419]/55`
- **Fix** : Tous bumped à `/72`

### 7. [`calendar/views/agenda-view.tsx`] — Heure fin + référence mission
- **Critère RGAA** : 3.2
- **Sévérité** : MINOR (texte 9-10px mono uppercase)
- **Cause** : `text-[10px] /55` (heure fin) + `text-[9px] /55` (référence)
- **Fix** : `/55` → `/72` (2 occurrences)

### 8. [`account/settings-row.tsx`] — Pas de focus-visible (button + Link)
- **Critère RGAA** : 10.7 (Prise de focus visible) / WCAG 2.4.7
- **Sévérité** : MAJEURE
- **Cause** : Composant utilisé sur 8 pages account, ni le branch `<button>` ni le branch `<Link>` n'avaient de `focus-visible:ring-*`
- **Fix** : Ajout `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-[#0F1419]/40 focus-visible:ring-offset-1` sur les deux branches

### 9. [`account/user-hero-card.tsx`] — Bouton hero sans focus visible
- **Critère RGAA** : 10.7
- **Sévérité** : MAJEURE
- **Cause** : `<button>` natif sans `focus-visible:*`
- **Fix** : Ajout du ring sur le `<button>` root

### 10. [`dossiers/import/wizard.tsx`] — Toggle pills sans focus visible
- **Critère RGAA** : 10.7
- **Sévérité** : MAJEURE
- **Cause** : 4 `<button aria-pressed>` (sélection logiciel source) sans focus visible
- **Fix** : Ajout focus ring

## Décisions design préservées (à valider par fondateur si évolution souhaitée)

- **Placeholders inputs** (`placeholder:text-[#0F1419]/40` ~3.2:1) : non corrigés ce lot — WCAG 1.4.11 (Non-Text Contrast 3:1) est respecté pour les composants UI, et les placeholders ne sont pas "essentiels" au sens WCAG. Néanmoins **recommandation** : passer à `/55` au prochain pass design pour s'aligner sur les bonnes pratiques 2026
- **Icônes décoratives** (`text-[#0F1419]/40` avec `aria-hidden`) : conforme — pas de contrainte de contraste pour les éléments décoratifs masqués des AT
- **Color-only signals** : aucun détecté sur les 28 pages. Status pills (Badge `variant="green/red/amber"`) ont toujours un libellé texte ("Conforme", "Expiré", "En cours"). Diagnostic chips (DPE/AMIANTE/etc.) ont le code couleur **+** le label texte
- **Modals & focus trap** : non audité ce lot (composants Sheet/Dialog Radix UI gèrent nativement le focus trap — confiance haute)
